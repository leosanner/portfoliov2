# Agent Guidelines — Portfolio v2

Full guidance for AI agents working in this repository.

## Language

- All files (code, comments, docs, specs, commit messages) → **English**
- Dialogue with the user → **Portuguese (PT-BR)**

## Project Context

Personal portfolio to showcase projects. Single admin authenticated via Google OAuth. Public visitors read only.

Full specs: `.specs/project/PROJECT.md` · `.specs/project/ROADMAP.md` · `.specs/project/STATE.md`

## Architecture

```
Cloudflare Pages (apps/web)   ←→   Cloudflare Workers (apps/api)
     React + Vite                        Hono + Hono RPC
          ↓                                     ↓
   Static assets (CDN)             D1 (SQLite) + Drizzle ORM
                                   Better Auth (Google OAuth)
```

Two independent Cloudflare deployments from one monorepo. CORS configured via `ALLOWED_ORIGIN` env var.

## Monorepo Structure

```
apps/
  web/          # React + Vite frontend
  api/          # Hono backend
packages/
  shared/       # Hono RPC route types, Zod schemas
.github/
  workflows/
    deploy-web.yml   # triggers on apps/web changes
    deploy-api.yml   # triggers on apps/api changes
.specs/
  project/      # PROJECT.md, ROADMAP.md, STATE.md
.claude/
  settings.json # Claude Code permissions
```

## TDD — Mandatory

Tests are written **before** implementation. No exceptions.

1. Write the test file (`.test.ts`) — it must fail
2. Implement until the test passes
3. Refactor if needed, keeping tests green

**Test runner:** Vitest (in both `apps/web` and `apps/api`)

Never write implementation code without a corresponding failing test first.

## Environment Variables

| Location | Purpose | Git |
|----------|---------|-----|
| `apps/api/.dev.vars` | Local dev secrets | Ignored |
| `apps/api/.dev.vars.example` | Contract — all required keys | Committed |
| Cloudflare dashboard | Preview + production secrets | N/A |
| GitHub Secrets | CI/CD injection | N/A |

**Rule:** Any new env var must be added to `.dev.vars.example` in the same PR.

## Database Environments

| Environment | D1 Database | When |
|-------------|------------|------|
| dev | Local Wrangler simulation | Local development |
| preview | `portfolio-preview` (remote) | PR branches |
| production | `portfolio-prod` (remote) | Main branch |

Migrations are generated by Drizzle Kit and applied automatically in CI before deploy.

```bash
pnpm db:migrate          # apply locally
pnpm db:generate         # generate migration from schema changes
```

## Permissions (what agents can do freely)

**Allowed without confirmation:**
- Filesystem: `ls`, `touch`, `mkdir`, `cp`, `mv`, `cat`, `head`, `tail`, `find`, `echo`, `pwd`
- Git: `status`, `diff`, `add`, `commit`, `log`, `push`, `pull`, `checkout`, `branch`, `merge`, `stash`, `fetch`, `init`
- Dev tools: `pnpm`, `npm`, `npx`, `wrangler`, `vitest`

**Requires explicit user confirmation:**
- `rm`, `rmdir`, or any destructive delete operation
- Force push (`git push --force`)
- Dropping or resetting databases
- Any irreversible operation

## Development Workflow

1. One increment at a time — implement, verify, then move forward
2. Each feature starts on a branch
3. PR → preview deploy (automatic, with migrations)
4. Merge to main → production deploy (automatic, with migrations)
5. Never skip the test phase to "move faster"

## Hono RPC Convention

Route types live in `packages/shared`. The frontend never makes raw `fetch` calls — always uses the typed Hono RPC client.

```ts
// packages/shared/src/routes.ts — define typed routes
// apps/web/src/lib/client.ts    — instantiate hono client
```

## Commit Style

```
feat: add project creation endpoint
fix: correct CORS header for preview environment
test: add unit tests for project route
chore: update drizzle schema for projects table
```

Atomic commits — one logical change per commit.
